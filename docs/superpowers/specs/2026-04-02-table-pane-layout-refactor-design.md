# TablePane Layout Refactor Design

- Date: 2026-04-02
- Target: `src/popup/panes/TablePane.tsx`
- Status: Approved in conversation

## Context

`サイト別機能` ペインは、`URLパターン` と `フォーカス維持` の設定を 1 画面に集約しているが、現状は説明文・入力欄・診断パネル・一覧がほぼ同じ重みで並んでいる。

そのため、次の課題がある。

- まず何を操作すべきかが見えにくい
- `フォーカス維持` の診断結果が入力フローに埋もれる
- `URLパターン` と `フォーカス維持` の役割差が見た目に出ていない

今回のリファクタでは、機能追加は行わず、情報の優先順位と見た目の階層を整理する。

## Goals

- `このタブで有効化` と現在状態を最初に把握できるようにする
- `URLパターン` と `フォーカス維持` を独立したカードとして整理する
- `フォーカス維持` では診断結果を主役として扱う
- 既存のダークテーマとアクア系アクセントを維持しながら、可読性を上げる
- 既存の機能、文言の意味、runtime の挙動は変えない

## Non-goals

- 新しい設定項目の追加
- Storage schema や runtime message の変更
- ナビゲーションレールや他ペインの全面改修
- 機能の意味を変える文言変更

## Chosen Direction

比較モック A `サマリー + セクションカード` 案を採用する。

この案では、上部に `現在のタブに対するアクションと状態` を集約し、その下に以下の 2 カードを縦に並べる。

1. `URLパターン`
2. `フォーカス維持`

選定理由:

- 現状構成からの差分が比較的小さく、実装コストと改善効果のバランスが良い
- `フォーカス維持` の診断結果を入力欄より前に出せる
- Chrome 拡張 popup の限られた幅でも無理が少ない

## Layout Structure

### 1. Header Summary

`pane-title` と主 CTA `このタブで有効化` を上段に置き、その直下にサマリー領域を追加する。

サマリー領域の役割:

- このペインで最初に注目すべき操作を示す
- 現在のタブに関する状態を簡潔に示す
- 下のカードに入る前に文脈を揃える

サマリー内に含める要素:

- 既存の補助説明を圧縮した短いテキスト
- 現在のタブ状態を示すピルまたは短い補足
- 必要に応じて、`フォーカス維持` の診断状態とつながる簡単なメッセージ

### 2. URL Pattern Card

`URLパターン` 用のカードを独立させ、次の順で構成する。

1. セクションタイトル
2. 1 行の補足説明
3. パターン入力行
4. 登録済み一覧または空状態

一覧の各行では、次の情報密度を揃える。

- 左: パターン文字列
- 右: 行フィルタスイッチ、削除ボタン

空状態は単純なテキストではなく、一覧領域と同じ枠内で薄いインセット表現にする。

### 3. Focus Override Card

`フォーカス維持` カードでは、診断パネルをカードの主役にする。

順序:

1. セクションタイトル
2. 補足説明
3. 診断パネル
4. パターン追加の補足
5. パターン入力行
6. 登録済み一覧または空状態

診断パネルでは次の要素を明確に分離する。

- 状態ピル
- 現在 URL
- 説明文
- 一致パターン
- `再診断`
- `このタブを再読み込み` (`reload-required` のときだけ)

これにより、ユーザーは `今どうなっているか` を先に読んだ上で、必要に応じて設定追加に進める。

## Visual Design

既存テーマトークンと色味は維持しつつ、視覚階層を強める。

### Principles

- 上部サマリーは、通常カードより少し強いアクセント背景で目線を集める
- 各カードは背景・境界線・内側余白を揃えて一貫性を出す
- 補助テキストは小さく、状態表示と操作を強く見せる
- 入力欄、一覧、空状態に共通の角丸とインセット感を持たせる

### Expected CSS Changes

- `TablePane` 専用の wrapper class を追加する
- サマリー帯用の新規クラスを追加する
- `pattern-list` / `pattern-item` / `empty-message` をカードレイアウトに合わせて調整する
- `focus-diagnostic-panel` の余白・背景・ボタン配置を再整理する
- モバイル幅相当でも横スクロールが出ないようにする

## Component-Level Changes

### JSX Changes

- `TablePane` のトップレベル構造を `header summary + stacked cards` に再編する
- 説明文の並びを簡略化し、補助テキストを各カード見出しの近くへ移動する
- `フォーカス維持` では診断パネルを入力欄より前に移動する
- 既存の button / input / switch / scroll area のコンポーネント利用は維持する

### State / Logic Changes

ロジック変更は原則行わない。

許容範囲:

- 表示位置変更に伴う JSX の並び替え
- 表示文脈に合わせた className の整理
- 必要最小限の派生表示テキストの追加

変更しないもの:

- storage read/write
- focus diagnostic resolution flow
- enable table sort action
- add/remove pattern flow
- add/remove focus pattern flow

## Testing Strategy

### Manual / Story Validation

- `TablePane` story で主 CTA、入力欄、空状態、一覧の視認性を確認する
- `フォーカス維持` の診断表示がカード上部で自然に読めるか確認する

### Automated Checks

- 既存の `TablePane` 周辺テストが落ちないことを確認する
- popup 関連のテストで見出しや主要導線が壊れていないことを確認する

想定コマンド:

```bash
pnpm vitest run tests/popup.table_sort_pane.test.ts
pnpm vitest run tests/e2e/popup.spec.ts
```

必要に応じて、レイアウト変更に合わせて既存アサーションを更新する。

## Risks

- 余白を増やしすぎると popup 高さで情報が折りたたまれて見えにくくなる
- `focus-diagnostic-panel` の並び替えで既存テストの文言前提が崩れる可能性がある
- 一覧項目のカード感を強くしすぎると、行フィルタスイッチとの視線競合が起こる

## Review Notes

- 比較モックで A 案を提示し、ユーザーが選択した
- ユーザー承認済みの設計は以下
  - 上部サマリー + 2 カード構成
  - `フォーカス維持` では診断を先頭へ
  - 既存テーマは維持しつつ階層を整理

## Implementation Exit Criteria

- `サイト別機能` ペインが `header summary + URLパターン card + フォーカス維持 card` 構成になっている
- 主 CTA と現在状態が初見で把握しやすい
- `フォーカス維持` の診断結果が入力欄に埋もれていない
- 既存機能と保存挙動は変わらない
- 関連テストまたは確認コマンドの結果を提示できる
